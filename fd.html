What accessible course design means

For web-based courses, accessibility means students can navigate and use your Brightspace shell, documents, and media with or without assistive technology. Typical practices include:

  • Using headings, lists, and semantic structure.
  • Providing alt text, captions, and transcripts.
  • Ensuring readable contrast and scalable text.
  • Designing so keyboard-only navigation is possible.
  • Offering multiple ways to access content and show learning.

WCAG 2.2 for Teaching POUR

WCAG 2.2 groups requirements into four principles: content must be perceivable, operable, understandable, and robust (POUR). Most higher education policies target Level AA.

Perceivable

  • Text alternatives for non-text content.
  • Captions and transcripts for time-based media.
  • Readable contrast and resizable text.

Operable

  • Keyboard access to all functionality.
  • Clear navigation and focus order.
  • No disruptive auto-play or flashing content.

Understandable

  • Clear, consistent language and layout.
  • Predictable navigation and labels in Brightspace.
  • Helpful instructions and error messages.

Robust

  • Clean HTML and document structure.
  • Compatible with screen readers and other tools.
  • Avoid dependence on one browser or plugin.

Making PDFs more accessible

  • Create content with headings and lists in Word or Google Docs first.
  • Use “Save as PDF” with tags enabled, not “Print to PDF”.
  • Run the accessibility checker in Word/Acrobat and fix issues.
  • In Brightspace, use the Accessibility Checker on HTML pages.

If a PDF is a scanned image, use OCR and consider replacing it with an accessible Brightspace page or an accessible file from the library.

Brightspace and Neurodiversity UDL

Neurodivergent students (for example ADHD, autism, dyslexia, anxiety) often benefit from predictable structure, clear cues, and flexible ways to engage. These are also strong accessibility practices.

Structure in Brightspace

  • Use consistent module names such as “Week 3 – Topic”.
  • Organize each module: Overview → Materials → Activities → Assessments.
  • Post a “Week at a Glance” page with expectations and due dates.
  • Use checklists for multi-step projects.
