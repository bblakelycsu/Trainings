What does “Perceivable” mean for my course?

Perceivable means that students can actually see, hear, or otherwise access the information in your course. If content is only visual, only audio, or only color-based, some students will miss it. WCAG 2.2 Section 1 focuses on:

  • Providing text alternatives for images, icons, and visual-only content.
  • Ensuring audio and video have captions and/or transcripts.
  • Structuring pages so assistive technologies can navigate them.
  • Making text and visuals distinguishable (contrast, color, spacing).

You do not need to be a web developer to make your course more accessible. Most changes happen in how you write your syllabus, create slides, set up Brightspace pages, and export PDFs. For technical adjustments (e.g., complex PDFs), eLearning and CTRE can support you.

Practical starting points:
  • Use descriptive alt text on images in Brightspace and PowerPoint.
  • Turn on and edit captions for lecture recordings.
  • Use heading styles in Word before converting to PDF.
  • Avoid relying only on color to convey correctness or emphasis.

If you feel stuck or unsure, email elearning@csu.edu with a sample file (syllabus, slide deck, or PDF), and we can walk through it with you.

1.1

Text Alternatives

WCAG 2.2: 1.1.1 Non-text Content (Level A)
Brightspace Word / PDF Level A
Show details and example

Provide meaningful text alternatives for any non-text content so that assistive technologies (like screen readers) can convey the same information. This includes photos, diagrams, icons, and decorative elements.

In Brightspace:
  • When you insert an image, use the Image Description / Alt Text field.
  • Describe the purpose (what the student needs to understand), not every visual detail.
  • Check “decorative image” only if the image adds no instructional meaning.
In Word / PDFs:
  • Right-click the image → Edit Alt Text → write a short, clear description.
  • When exporting to PDF, use “Best for electronic distribution and accessibility.”
  • Run the built-in Accessibility Checker in Word or Adobe Acrobat.
Example to analyze

A bar chart in your syllabus shows exam averages for three sections: Section A (72%), Section B (80%), Section C (88%). The current alt text says: “Bar chart of exam scores.”

Question: Does this alt text clearly convey the important information to a student who cannot see the chart?

Better alt text might be: “Bar chart showing average exam scores by section: Section A 72%, Section B 80%, Section C 88%, with Section C scoring the highest.”
1.2

Time-Based Media (Audio and Video)

WCAG 2.2: 1.2.x (Captions, Audio Descriptions, Transcripts)
Lecture Capture Linked Media Level A / AA
Show details and example

Students who are Deaf/hard of hearing, have auditory processing needs, or are in noisy environments may not be able to follow audio-only content. WCAG requires captions for video and transcripts for audio-only content.

For your recorded lectures:
  • Turn on captions in your recording tool or platform (e.g., YuJa, Zoom, Teams).
  • Review and edit auto-generated captions for accuracy in names, terminology, and numbers.
  • Provide a downloadable transcript for audio-only materials when feasible.
In Brightspace:
  • Embed videos with captions enabled by default.
  • Make sure any linked media mentions “captions available” in the link text.
Example to analyze

You recorded a 25-minute lecture and uploaded it to Brightspace. Auto-captions exist but you never reviewed them. Several technical terms and student names are incorrect.

Question: Does this meet accessibility expectations? What should you do next?

Auto-captions alone do not fully meet best practices. You should open the caption editor, correct errors in key terminology, and save updated captions. If students report issues, offer a transcript or alternative resource as needed.
1.3

Adaptable Structure

WCAG 2.2: 1.3.x (Info and Relationships)
Content Pages Syllabi / Handouts Level A / AA
Show details and example

Adaptable means your content preserves structure when viewed in different ways (screen readers, mobile devices, high zoom). You achieve this by using true headings, lists, and table markup so assistive technologies can interpret the relationships.

In Word / PDFs:
  • Use built-in heading styles for titles and section headings (Heading 1, Heading 2, etc.).
  • Use proper bullet/numbered list tools instead of typing dashes manually.
  • Avoid using tables for layout; reserve them for real data relationships.
In Brightspace:
  • Use heading styles in the HTML editor to organize long content pages.
  • Keep a logical order: page title → overview → sections → links/resources.
Example to analyze

A syllabus is formatted by increasing font size and bolding lines to “look like” headings, but no heading styles are used. When a screen reader user navigates, they cannot jump between sections.

Question: What is one simple change you can make in Word before exporting to PDF?

Apply built-in heading styles (e.g., Heading 1 for course title, Heading 2 for major sections, Heading 3 for subsections). This allows screen readers and navigation tools to move between sections quickly.
1.4

Distinguishable Content

WCAG 2.2: 1.4.x (Contrast, Use of Color, Visual Presentation)
Visual Design Slides / Handouts Level AA (mostly)
Show details and example

Distinguishable means students can clearly see and read content. This includes sufficient contrast, reasonable text size, and not using color alone to communicate meaning.

Your course visuals should:
  • Use text colors that strongly contrast with the background (e.g., dark text on white).
  • Avoid putting important text directly on busy images.
  • Never rely on color alone to show correct/incorrect, required/optional, or status.
Quick checks:
  • If you print a slide in black and white, can you still tell what is important?
  • At 125–150% zoom, is text still legible without horizontal scrolling?
Example to analyze

A quiz review slide shows correct answers in green and incorrect answers in red, with no labels or icons. Students with color blindness cannot distinguish them.

Question: How can you redesign this so that color is not the only cue?

Add text labels such as “Correct” and “Incorrect” next to each answer and consider using icons (e.g., check mark / X) or patterns. Keep color as a helpful extra cue, not the only one.
